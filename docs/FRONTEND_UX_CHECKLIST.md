# Frontend Agent UX 设计检查清单

## 视觉设计 (Visual Design)

### 配色方案
- [ ] 主色调符合金融科技主题（深蓝、紫色、琥珀色）
- [ ] 配色对比度符合 WCAG 2.1 AA 标准（文本≥4.5:1）
- [ ] 状态色语义清晰（成功=绿，警告=黄，错误=红）
- [ ] 深色模式配色经过优化，无过亮/过暗区域
- [ ] 渐变色使用适度，不喧宾夺主

### 排版设计
- [ ] 字体层级清晰（h1 > h2 > h3 > body）
- [ ] 正文行高适宜（1.5-1.7）
- [ ] 段落间距合理，阅读舒适
- [ ] 数字使用等宽字体便于对比
- [ ] 中文文案使用合适的中文字体

### 视觉层次
- [ ] 重要信息通过大小/颜色/位置突出
- [ ] 卡片、分区边界清晰
- [ ] 留白适度，不拥挤也不空旷
- [ ] 图标使用一致，风格统一

---

## 交互设计 (Interaction Design)

### 点击反馈
- [ ] 按钮悬停效果明显（颜色变化/阴影）
- [ ] 按钮点击有按压效果
- [ ] 链接有明确的下划线或颜色区分
- [ ] 卡片可点击时有视觉提示

### 加载状态
- [ ] 数据加载时显示骨架屏或 Loading 动画
- [ ] 加载时间>3s 时显示进度提示
- [ ] 加载完成有明确的视觉反馈
- [ ] 加载失败有友好的错误提示

### 动画效果
- [ ] 页面过渡流畅（200-300ms）
- [ ] 动画缓动函数自然（ease-out/ease-in-out）
- [ ] 无突兀的闪烁或跳动
- [ ] 动画性能良好（60fps）

---

## 响应式设计 (Responsive Design)

### 移动端 (<640px)
- [ ] 单列布局，内容垂直排列
- [ ] 触摸目标≥44px
- [ ] 字体大小≥14px
- [ ] 导航可折叠或底部固定
- [ ] 横向滚动有明确提示

### 平板端 (640px-1024px)
- [ ] 双列或三列布局
- [ ] 卡片网格自适应
- [ ] 表格可横向滚动
- [ ] 侧边栏可收起

### 桌面端 (>1024px)
- [ ] 充分利用宽屏空间
- [ ] 多列布局展示更多信息
- [ ] 固定导航或侧边栏
- [ ] 大图大表清晰展示

---

## 无障碍设计 (Accessibility)

### 颜色对比度
- [ ] 普通文本对比度≥4.5:1
- [ ] 大文本对比度≥3:1
- [ ] 图表颜色可区分（考虑色盲用户）
- [ ] 不单纯依赖颜色传达信息

### 键盘导航
- [ ] 所有交互元素可 Tab 访问
- [ ] 焦点顺序合理
- [ ] 焦点状态清晰可见
- [ ] 提供跳过导航功能

### 屏幕阅读器
- [ ] 图片有 alt 描述
- [ ] 表单有 label 关联
- [ ] 图表有文字摘要
- [ ] 使用语义化 HTML 标签
- [ ] 动态内容使用 aria-live 通知

---

## 信息架构 (Information Architecture)

### 数据展示
- [ ] 关键指标（净值、收益率）醒目展示
- [ ] 数据更新時間清晰可见
- [ ] 复杂数据分层展示
- [ ] 提供数据筛选和排序功能

### 图表设计
- [ ] 图表类型适合数据特点
- [ ] 图例清晰易懂
- [ ] 坐标轴标签完整
- [ ] 支持缩放和悬停查看

### 导航设计
- [ ] 当前位置清晰指示
- [ ] 导航层级不超过 3 层
- [ ] 提供面包屑或返回功能
- [ ] 搜索功能易于发现

---

## 性能体验 (Performance UX)

### 加载性能
- [ ] 首屏内容 3 秒内显示
- [ ] 可交互时间<5 秒
- [ ] 大数据列表虚拟滚动
- [ ] 图片和图表懒加载

### 渲染性能
- [ ] 滚动流畅无卡顿
- [ ] 动画帧率>60fps
- [ ] 无明显的布局抖动
- [ ] 内存使用合理

### 等级对应
- **A (90-100%)**: 优秀，可直接发布
- **B (75-89%)**: 良好，少量优化
- **C (60-74%)**: 合格，需要优化
- **D (40-59%)**: 较差，大量优化
- **F (<40%)**: 不可接受，需要重做
