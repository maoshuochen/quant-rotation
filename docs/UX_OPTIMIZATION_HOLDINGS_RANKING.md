# 前端 UX 优化方案 - 持仓与排名合并

## 问题分析

### 当前结构

```
ranking.json
├── ranking: [19 个指数]
    ├── 前 5 名 → 在"建议持仓"卡片展示
    └── 全部 19 个 → 在"全部排名"卡片展示
```

**问题**:
1. 前 5 名重复展示两次
2. 用户需要展开两个卡片查看相同因子数据
3. 信息密度低，需要更多滚动

### 当前数据结构

```json
{
  "ranking": [
    {
      "code": "399986.SZ",
      "name": "银行指数",
      "rank": 1,
      "score": 0.6617,
      "factors": {...},
      "attribution": {...}
    },
    // ... 共 19 个
  ]
}
```

## 优化方案

### 方案 A: 统一排名列表（推荐）

**布局**:
```
┌─────────────────────────────────────┐
│ 📊 全部排名（19 个指数）              │
├─────────────────────────────────────┤
│ 🏆 TOP 1  银行指数     得分 0.662  [展开] │
│    └─ 因子得分 + 归因详情              │
│                                      │
│ 🥈 TOP 2  煤炭指数     得分 0.642  [展开] │
│ 🥉 TOP 3  新能源指数   得分 0.634  [展开] │
│                                      │
│  4. 科技指数        得分 0.550  [展开] │
│  5. 医药指数        得分 0.518  [展开] │
│  6. 证券指数        得分 0.508  [展开] │
│  ...                                 │
└─────────────────────────────────────┘
```

**视觉区分**:
- 前 3 名：使用金/银/铜色徽章，背景高亮
- 4-5 名：普通徽章，轻微强调
- 6 名后：简化的行内展示

**数据结构修改**:
```json
{
  "ranking": [
    {
      "code": "399986.SZ",
      "name": "银行指数",
      "rank": 1,
      "score": 0.6617,
      "is_top_pick": true,
      "factors": {...},
      "attribution": {...}
    },
    // ...
  ]
}
```

### 方案 B: 分组展示

```
┌─────────────────────────────────────┐
│ 🏆 建议持仓（前 5 名）                  │
├─────────────────────────────────────┤
│ 1. 银行指数    得分 0.662  [展开详情] │
│ 2. 煤炭指数    得分 0.642  [展开详情] │
│ 3. 新能源指数  得分 0.634  [展开详情] │
│ 4. 科技指数    得分 0.550  [展开详情] │
│ 5. 医药指数    得分 0.518  [展开详情] │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 📋 其他排名（第 6-19 名）              │
├─────────────────────────────────────┤
│ 6. 证券指数    得分 0.508  [展开]    │
│ 7. 消费指数    得分 0.495  [展开]    │
│ ...                                 │
└─────────────────────────────────────┘
```

**优点**:
- 保留分组概念，用户理解成本低
- 减少重复，前 5 名只展示一次
- 降低滚动距离

## 推荐实施步骤

### 1. 修改数据生成脚本

`scripts/generate_web_data.py`:
```python
# 添加 is_top_pick 标记
for i, row in ranking_df.iterrows():
    item = {
        'code': code,
        'name': idx_info.get('name', code),
        'rank': int(row['rank']),
        'score': round(float(row['total_score']), 4),
        'is_top_pick': row['rank'] <= 5,  # 新增标记
        'factors': factors,
        'attribution': {...}
    }
```

### 2. 修改前端组件

**Dashboard.jsx**:
```jsx
// 移除 HoldingCard 组件，统一使用 RankingCard
const UnifiedRankingCard = ({ data, activeFactors }) => {
  const [expanded, setExpanded] = useState(null)
  
  return (
    <div className="space-y-2">
      {data.ranking.map((item, idx) => (
        <RankingListItem
          key={item.code}
          item={item}
          isExpanded={expanded === item.code}
          onToggle={() => setExpanded(expanded === item.code ? null : item.code)}
          isTopPick={item.rank <= 5}
          activeFactors={activeFactors}
        />
      ))}
    </div>
  )
}
```

### 3. 视觉优化

```css
/* 前三名高亮 */
.rank-1 { badge: 'bg-amber-500'; bg: 'bg-amber-500/10'; }
.rank-2 { badge: 'bg-zinc-400'; bg: 'bg-zinc-400/10'; }
.rank-3 { badge: 'bg-amber-700'; bg: 'bg-amber-700/10'; }

/* 4-5 名强调 */
.rank-4, .rank-5 { badge: 'bg-white/10'; border: 'border-amber-500/20'; }

/* 6 名后简化 */
.rank-6+ { layout: 'compact'; }
```

## 用户故事

**场景**: 用户想了解当前推荐持仓和完整排名

**优化前流程**:
1. 查看"建议持仓"卡片 → 展开查看因子
2. 滚动到"全部排名" → 再次查看相同数据
3. 需要 mentally merge 两个列表

**优化后流程**:
1. 查看统一的排名列表
2. 前 3 名自动高亮
3. 展开任意项目查看详情
4. 一屏内获取完整信息

---

*建议采用方案 A（统一排名列表），最简洁且用户体验最佳*
