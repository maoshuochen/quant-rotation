# 指数轮动量化系统 (Quant Rotation System)

基于多因子评分的指数轮动研究系统，支持 A 股主要宽基指数和行业指数的评分、排名、信号生成、回测和前端可视化。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements/base.txt
```

### 2. 配置参数

编辑 `config/universe.yaml`、`config/strategy.yaml` 和 `config/runtime.yaml` 配置指数池、策略参数与运行设置。

### 3. 运行

```bash
# 每日评分与调仓信号
python scripts/daily_run_baostock.py

# 回测
python scripts/backtest_baostock.py

# 生成前端数据
python scripts/generate_data.py
```

## 因子体系

当前正式主模型使用 3 个主因子参与总分，权重来自 `config/strategy.yaml`：

| 因子 | 权重 | 内容 |
|------|------|------|
| 强度 | 23.5% | `50%` 绝对动量 + `50%` 相对强弱 |
| 趋势 | 18% | 价格相对 MA20 / MA60 的位置 |
| 资金流 | 58.5% | 成交量趋势、量价配合、成交金额、资金活跃强度 |

可以先把这套模型理解成一句话：

“先找真正强的方向，再确认趋势没有走坏，最后优先选择有量价配合、资金活跃度更高的标的。”

更具体一点，3 个主因子各自回答的是 3 个不同问题：

- `strength`：这个方向是不是强者。
  既看它自己涨得强不强，也看它相对大盘有没有超额优势。
- `trend`：这个方向现在是不是还站在上升结构里。
  避免只因为过去涨过，就去追已经转弱的标的。
- `flow`：这个方向最近有没有真实的量价活跃度在支撑。
  不是只看价格表面上涨，而是看背后有没有“交易在发生、关注在提升、资金在承接”。

因此这 3 个因子在策略里的分工可以记成：

- `strength` 负责“找强者”
- `trend` 负责“看结构”
- `flow` 负责“做确认”

当前正式基线里，`flow` 权重最高，不是因为它单独最神奇，而是因为前面的实验表明：

- `strength` 已经吸收了大量价格强度信息
- `trend` 提供的是结构过滤
- `flow` 与前两者重叠更少，增量信息更多

补充说明：

- `动量`、`相对强弱`、`估值`、`波动` 仍会在评分与归因中计算，但当前只有合成后的 `strength` 计入主模型总分。
- 市场状态切换会影响看板展示和动态解释，但当前基线回测主分仍以上述 3 因子为核心。
- 当前风控基线只保留 `8%` 移动止损，不再使用固定现金缓冲。

### 强度因子公式

当前正式基线的 `strength` 因子把绝对动量和相对强弱合成到一个主因子里：

```text
Strength =
0.50 * Momentum_score
+ 0.50 * RelativeStrength_score
```

它的作用是把“绝对强”和“相对强”合成一个统一的强度分数。

- `Momentum_score` 看的是：最近一段时间你自己涨得强不强
- `RelativeStrength_score` 看的是：你相对沪深 300 有没有跑赢

这两者放在一起，比单独只看动量更稳一些，因为它能区分：

- “只是跟着市场一起涨”
- “自己本身就是市场里更强的那一批”

如果用更口语的方式说：

`strength` 不是单看“涨了没”，而是看“是不是一个真正有领先性的强方向”。

### 资金流因子公式

当前正式基线的 `flow` 因子只使用 4 个基础量价子分数，分数范围统一压到 `0 ~ 1`：

```text
Flow =
0.35 * S_vol
+ 0.10 * S_corr
+ 0.35 * S_amt
+ 0.20 * S_intensity
```

其中：

- `S_vol`：成交量趋势分。比较最近 20 日平均成交量与前 20 日平均成交量，放量越明显分数越高。
- `S_corr`：量价配合分。计算价格收益率与成交量变化率的相关性，越同步越高分。
- `S_amt`：成交金额趋势分。比较最近 20 日平均成交额与前 20 日平均成交额；若金额数据缺失，则退化为 `S_vol`。
- `S_intensity`：活跃强度分。统计最近 20 天中，有多少天成交量高于近 60 天成交量中位数。

它实际在看 4 件事：

- 最近是不是在放量
- 价格和成交量是不是同步
- 成交金额有没有一起扩张
- 高活跃交易日是不是持续变多

可把它理解成一句话：`flow` 评估的不是“涨了没”，而是“最近有没有持续、健康、可验证的量价活跃度在推动这个方向”。

从实验结果看，当前模型更重视：

- 放量
- 成交金额扩张

而弱化了量价相关性这一项的影响。考虑到北向资金与 ETF 份额覆盖不稳定，它们已从正式基线中移除，不再参与 `flow` 打分。

### 相对强弱底层公式

`relative_strength` 衡量的是标的相对基准的超额收益。当前默认使用沪深 300 作为基准，优先使用 `60` 日窗口，数据不足时降级为 `20` 日或 `10` 日窗口。

```text
RS_raw = R_index - R_benchmark

R_index = P_t / P_(t-k) - 1
R_benchmark = B_t / B_(t-k) - 1

RS_score = clip(0.5 + RS_raw, 0, 1)
```

其中：

- `P_t`：标的当前价格
- `P_(t-k)`：标的 `k` 日前价格
- `B_t`：基准当前价格
- `B_(t-k)`：基准 `k` 日前价格

它和动量的区别在于：

- `momentum` 问的是：你自己涨了多少
- `relative_strength` 问的是：你比市场平均强多少

所以它更适合用来回答：

“这是不是一个真正跑赢大盘的方向，而不是只在市场普涨时被动跟涨？”

### 趋势因子直觉

`trend` 不复杂，它本质上是在做结构过滤。

当前实现里主要看两件事：

- 当前价格是否站上 `MA20`
- 当前价格是否站上 `MA60`

如果两个都站上，趋势分更高；如果跌回均线下方，趋势分就会下降。

它的意义不是“找最猛的”，而是：

“已经很强的方向，现在是不是还维持在一个健康的上升结构里。”

## GitHub Actions 自动化

数据刷新工作流每天 UTC 19:00（北京时间凌晨 3 点）自动运行：
1. 获取最新 ETF 数据
2. 运行回测更新
3. 生成 `web/public/*.json`

前端同步工作流在 `main` 分支的 `web/**` 变更后自动执行，负责构建前端并更新 `site/`。
