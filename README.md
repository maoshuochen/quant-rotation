# 指数轮动量化系统 (Quant Rotation System)

基于多因子评分的指数轮动研究系统，支持 A 股主要宽基指数和行业指数的评分、排名、信号生成、回测和前端可视化。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements/base.txt
```

### 2. 配置参数

编辑 `config/universe.yaml`、`config/strategy.yaml` 和 `config/runtime.yaml` 配置指数池、策略参数与运行设置。

### 3. 运行

```bash
# 每日评分与调仓信号
python scripts/daily_run_baostock.py

# 回测
python scripts/backtest_baostock.py

# 生成前端数据
python scripts/generate_data.py
```

## 因子体系

当前正式主模型使用 4 个主因子参与总分，权重来自 `config/strategy.yaml`：

| 因子 | 权重 | 内容 |
|------|------|------|
| 动量 | 13.5% | 近 6 个月收益率为主，辅以近 1 个月变化做归因展示 |
| 趋势 | 18% | 价格相对 MA20 / MA60 的位置 |
| 资金流 | 58.5% | 成交量趋势、量价配合、成交金额、资金活跃强度 |
| 相对强弱 | 10% | 相对沪深 300 的超额收益 |

补充说明：

- `估值`、`波动`、`相对强弱` 仍会在评分与归因中计算，但当前不计入主模型总分。
- 市场状态切换会影响看板展示和动态解释，但当前基线回测主分仍以上述 3 因子为核心。
- 当前风控基线只保留 `8%` 移动止损，不再使用固定现金缓冲。

### 资金流因子公式

当前正式基线的 `flow` 因子只使用 4 个基础量价子分数，分数范围统一压到 `0 ~ 1`：

```text
Flow =
0.35 * S_vol
+ 0.10 * S_corr
+ 0.35 * S_amt
+ 0.20 * S_intensity
```

其中：

- `S_vol`：成交量趋势分。比较最近 20 日平均成交量与前 20 日平均成交量，放量越明显分数越高。
- `S_corr`：量价配合分。计算价格收益率与成交量变化率的相关性，越同步越高分。
- `S_amt`：成交金额趋势分。比较最近 20 日平均成交额与前 20 日平均成交额；若金额数据缺失，则退化为 `S_vol`。
- `S_intensity`：活跃强度分。统计最近 20 天中，有多少天成交量高于近 60 天成交量中位数。

可把它理解成一句话：`flow` 评估的不是“涨了没”，而是“最近有没有持续、健康、可验证的量价活跃度在推动这个方向”。

也就是说，当前模型更重视“放量”和“成交金额扩张”，弱化了量价相关性这一项的影响。考虑到北向资金与 ETF 份额覆盖不稳定，它们已从正式基线中移除，不再参与 `flow` 打分。

### 相对强弱因子公式

`relative_strength` 衡量的是标的相对基准的超额收益。当前默认使用沪深 300 作为基准，优先使用 `60` 日窗口，数据不足时降级为 `20` 日或 `10` 日窗口。

```text
RS_raw = R_index - R_benchmark

R_index = P_t / P_(t-k) - 1
R_benchmark = B_t / B_(t-k) - 1

RS_score = clip(0.5 + RS_raw, 0, 1)
```

其中：

- `P_t`：标的当前价格
- `P_(t-k)`：标的 `k` 日前价格
- `B_t`：基准当前价格
- `B_(t-k)`：基准 `k` 日前价格

可以把它理解成一句话：`relative_strength` 看的不是“你涨了多少”，而是“你有没有持续跑赢市场基准”。

## GitHub Actions 自动化

数据刷新工作流每天 UTC 19:00（北京时间凌晨 3 点）自动运行：
1. 获取最新 ETF 数据
2. 运行回测更新
3. 生成 `web/public/*.json`

前端同步工作流在 `main` 分支的 `web/**` 变更后自动执行，负责构建前端并更新 `site/`。
